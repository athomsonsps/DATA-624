# DATA 624 Group 3 Project #1 and Bonus 1 and Bonus 2 

## Executive Summary

This project uses time series forecasting to support business planning for Lone Star Appliances and Texas HealthLink. The main goal is not just to create forecasts, but to explain how the forecasts can support real decisions.

Exercise #1 focuses on Lone Star Appliances. The forecast estimates quarterly appliance shipments and supports production, staffing, inventory, logistics, and risk planning. The analysis looks at shipment growth, seasonal patterns, the 2020 disruption, model comparison, and final forecast recommendations.

Exercise #2 focuses on Texas HealthLink. The forecast estimates monthly patient visits for five clinics and the full system. The analysis supports staffing, budgeting, scheduling, facility planning, and systemwide resource allocation.

Bonus #1 compares the R forecasting workflow with a Python forecasting workflow using the same Texas HealthLink data. The Python work tests ETS, SARIMA, and regression with monthly seasonality. It uses the last 12 months as a holdout period, selects the best Python model by clinic, and produces 2024 clinic-level and systemwide forecasts.

Bonus #2 explains time series forecasting concepts in a clear interview-ready format. It includes an executive summary, graph commentary, and mock interview questions and answers in plain language.

## Exercise #1: Lone Star Appliances

### Board-Level Summary

### Technical Work

## Exercise #2: Texas HealthLink

### Board-Level Summary

This section explains the forecasting problem for Texas HealthLink and why clinic-level planning matters. The focus is on patient visits, staffing, budgeting, scheduling, and clinic capacity.

### Clinic-Level Forecasts

Each clinic is forecasted separately because demand is not the same across all locations. This helps leadership make better staffing and scheduling decisions.

### Systemwide Forecast

The systemwide forecast gives leadership a total view of expected patient volume. It supports budgeting, supply planning, and high-level resource decisions.

### Model Comparison

The models are compared using accuracy measures and holdout validation. This helps avoid choosing a model based only on appearance.

### Recommendation

Texas HealthLink should use both clinic-level forecasts and the systemwide forecast. The clinic forecasts should guide staffing and scheduling, while the systemwide forecast should guide budgeting and resource planning.

## Bonus #1: Python vs. R Forecasting Comparison

### Executive Summary

Bonus #1 compares R and Python for the Texas HealthLink forecasting problem. The goal is to see whether Python can produce comparable forecasts and whether it adds value to the forecasting workflow.

### Python Workflow

The Python script uses the same HealthLink data as the R workflow. It tests ETS, SARIMA, and regression with monthly seasonality. The last 12 months are used as a holdout period.

### Python Model Results

The best Python model is selected separately for each clinic based on holdout accuracy. This is important because one model is not always best for every clinic.

### Python Systemwide Forecast

The systemwide Python forecast is created using a bottom-up method. Each clinic is forecasted first, and then the clinic forecasts are summed to create the total system forecast.

### R vs. Python Comparison

R is stronger for direct time series analysis, reporting, diagnostics, and classroom-style forecasting work. Python is stronger when the forecast needs to connect with automation, dashboards, machine learning, or production systems.

### Recommendation

For this project, R should remain the main forecasting tool because it is easier to explain and audit. Python should be used as a second tool when the forecasting workflow needs to scale or connect to other systems.

The main lesson from this project is that forecasting is useful when it supports a decision. Lone Star Appliances can use the forecast for production, staffing, inventory, and logistics planning.
